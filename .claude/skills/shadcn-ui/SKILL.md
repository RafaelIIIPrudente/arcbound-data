# shadcn/ui (stack alignment)

Keep this repo's UI aligned with **shadcn/ui** as this repo uses it ‚Äî **new-york** style, **CSS
variables**, **Tailwind v4**, **RSC**, on **Radix** primitives. The distilled, cited docs live in
[`references/shadcn-ui-docs.md`](references/shadcn-ui-docs.md); the wiring is in
[`components.json`](../../../components.json), [`src/components/ui/`](../../../src/components/ui), and
[`src/lib/utils.ts`](../../../src/lib/utils.ts). Tokens/theme are the [`tailwind-v4`](../tailwind-v4/SKILL.md)
skill's job ‚Äî this skill is about the component layer.

## When to use

- Adding a new primitive (`pnpm dlx shadcn@latest add <name>`) or composing an existing one.
- Building a feature component (a form, table, dialog, dropdown) out of `ui/*` primitives.
- Reviewing a diff that touches `src/components/ui/*`, `components.json`, or a `cva` variant map.
- Any time you're tempted to reach for another UI kit, hand-edit a vendored `ui/*` file, or add a
  Base-UI idiom.

## Pinned version

From `package.json`: `radix-ui ^1.6.2`, `@radix-ui/react-slot ^1.1.2`,
`class-variance-authority ^0.7.1`, `clsx ^2.1.1`, `tailwind-merge ^3.0.1`, `lucide-react ^0.469.0`,
`next-themes ^0.4.4`, `tw-animate-css ^1.2.4`. shadcn config is **new-york / cssVariables / Tailwind
v4 / RSC** (`components.json`). **The live shadcn docs are migrating to Base UI ‚Äî this repo stays on
Radix.** Do **not** adopt Base UI idioms (`render`/`nativeButton`, `--base base`) on this pin (see the
Refresh section).

## Current idioms (new-york / Radix)

Sourced + cited in [`references/shadcn-ui-docs.md`](references/shadcn-ui-docs.md):

- **Components are owned code**, not an installed library ‚Äî copied in via the CLI, built on Radix,
  styled with `cva`, composed with `cn()`, made polymorphic with `asChild`/Slot. You edit them.
- **Add primitives with the CLI:** `pnpm dlx shadcn@latest add <name>` (the package is `shadcn`, not
  the deprecated `shadcn-ui`). Import from `@/components/ui/<name>`.
- **`cva` for variants:** a `cva(base, { variants, defaultVariants })` map + `VariantProps<typeof x>`
  for the prop types; export the variants fn so an `<a>`/`<Link>` can borrow the style
  (`className={cn(buttonVariants({ variant, size }))}`).
- **`cn()` everywhere:** `twMerge(clsx(...))` ‚Äî assemble conditional classes, then de-conflict so a
  caller's `className` overrides the default.
- **Tailwind-v4 shadcn baseline:** `data-slot` attributes, **no `forwardRef`** (plain function
  components + ref-as-prop), `size-*` utilities, semantic CSS-variable theming, `tw-animate-css`.
- **Dark mode via `next-themes`** with `attribute="class"` (drives the `.dark` selector).

## This repo's conventions

- **`components.json`** ‚Äî `style: "new-york"`, `rsc: true`, `tsx: true`, `tailwind.config: ""` (no JS
  config ‚Äî Tailwind v4), `cssVariables: true`, `baseColor: "neutral"`, `iconLibrary: "lucide"`, and
  the `@/components` ¬∑ `@/components/ui` ¬∑ `@/lib` ¬∑ `@/lib/utils` aliases. `baseColor` is fixed at
  init ‚Äî don't try to change it.
- **`src/components/ui/*`** ‚Äî the vendored primitives (`button`, `badge`, `card`, `input`, `table`,
  `form`, `select`, `dialog`, `dropdown-menu`, `chart`, `sidebar`, ‚Ä¶). `button.tsx` is the reference
  pattern: `cva` `buttonVariants`, `import { Slot } from "radix-ui"` ‚Üí `Slot.Root` when `asChild`,
  `data-slot="button"`, prop type `React.ComponentProps<"button"> & VariantProps<‚Ä¶> & { asChild? }`.
- **`src/lib/utils.ts`** ‚Äî the one `cn()` helper (`twMerge(clsx(inputs))`). Compose all classes through
  it; never string-concat classNames.
- **`src/components/theme/theme-provider.tsx` + `mode-toggle.tsx`** ‚Äî the `next-themes` provider
  (`attribute="class"`) and the light/dark toggle; **`src/components/ui/sonner.tsx`** is the toast
  surface wired to the theme.
- **Feature components** live under `src/components/dashboard/*` and are built **from** `ui/*`
  primitives (see the Customers reference: `customer-form.tsx`, `customers-table.tsx`,
  `status-badge.tsx`) ‚Äî they don't reinvent primitives.
- **`AGENTS.md`** ‚Äî "UI is shadcn/ui + Tailwind. Add primitives with `pnpm dlx shadcn@latest add
<name>`. Compose with `cn()`. No other UI kit."

## Banned / outdated

- **No competing UI kit** (MUI, Chakra, Ant, Mantine, ‚Ä¶) ‚Äî shadcn/ui + Tailwind only.
- **Don't gratuitously hand-edit vendored `ui/*` files** ‚Äî they're regenerated by the CLI and are
  **coverage-excluded** (`src/components/ui/**` in `vitest.config.ts`) / untested by design. Prefer
  wrapping/composing; edit a primitive only with intent, and expect a re-`add` to overwrite it.
- **No `shadcn-ui` package** (deprecated) ‚Äî use `shadcn@latest`.
- **No Base UI idioms on this pin** ‚Äî the live docs show `<Button render={<a />} nativeButton={false}>`
  and a `--base base` CLI; this repo is Radix, so use **`asChild` + Slot** and classic neutral base
  colors instead.
- **No hand-sorted classes** ‚Äî `prettier-plugin-tailwindcss` owns class order (`pnpm format`).

## Common tasks

- **Add a primitive:** `pnpm dlx shadcn@latest add <name>`, then compose it in a feature component ‚Äî
  don't author Radix wrappers by hand when the CLI has one.
- **A styled `<Link>`/`<a>`:** borrow a variant map ‚Äî `className={cn(buttonVariants({ variant: "ghost"
}))}` ‚Äî instead of duplicating classes.
- **A variant on an owned component:** extend its `cva` map (`variants` + `defaultVariants`), keep
  `VariantProps<typeof ‚Ä¶>` as the prop source of truth.
- **A polymorphic trigger:** pass `asChild` and render the child (e.g. `DropdownMenuTrigger asChild`
  wrapping a `Button`) rather than nesting interactive elements.
